OpenClaw: быстрая инструкция по запуску и защите сервера

Небольшая страница-памятка для ролика: подготовка окружения, проброс локального порта, первичная настройка и базовое обезопашивание сервера.

Тема оформления

По умолчанию страница подхватывает системную тему устройства.

1

Подготовка Docker и окружения

Сначала скачайте и выполните стартовый скрипт. Он подготовит сервер для дальнейшей настройки OpenClaw.

wget -O /tmp/prepare-openclaw.sh https://amorev.ru/misc/openclaw/prepare-openclaw-1.sh && sudo bash /tmp/prepare-openclaw.sh
2

Поднимите SSH-туннель

В отдельной вкладке терминала сделайте проброс локального порта 8000 на удалённый сервер.

ssh -L 8000:127.0.0.1:8000 root@<ВАШ IP адрес>

После этого откройте в браузере адрес http://localhost:8000 и заполните в env.txt свои значения.

3

Обезопасьте сервер

После первичной настройки выполните скрипт защиты. Он применяется отдельным шагом после подготовки.

wget -qO- https://amorev.ru/misc/openclaw/prepare-openclaw-2.sh | sudo bash
4

Проверьте вход под новым пользователем

Убедитесь, что всё в порядке: зайдите под именем пользователя и паролем, который вы настроили.

ssh openclaw@<ВАШ_IP_СЕРВЕРА> -p 2091
5

Установите OpenClaw обычным способом

Это стандартный сценарий установки без дополнительных предупреждающих режимов. Просто выполните следующую команду.

wget -qO- https://amorev.ru/misc/openclaw/prepare-openclaw-3.sh | sudo bash

Дождитесь появления в консоли строки ==> Done.

6

Режим WARNING

Этот шаг вынесен отдельно как предупреждающий сценарий. Используйте его только если вам действительно нужен именно такой режим запуска.

wget https://amorev.ru/misc/openclaw/prepare-openclaw-4-WARNING.sh -O setup.sh && bash setup.sh

После завершения также ориентируйтесь на появление строки ==> Done.

Краткий чеклист

  • Подготовить сервер стартовым скриптом.
  • Сделать SSH-туннель на локальный порт 8000.
  • Открыть localhost:8000 и заполнить env.txt.
  • Запустить скрипт обезопашивания.
  • Проверить доступ под пользователем openclaw.

Важно

Все команды запускайте внимательно и только на том сервере, который действительно предназначен для OpenClaw.

Полный автономный режим стоит включать только если вы понимаете последствия и ограничения такого сценария.
